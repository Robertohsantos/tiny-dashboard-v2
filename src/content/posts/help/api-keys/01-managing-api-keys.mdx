## Managing Your API Keys

The API Keys page is your central hub for managing authentication keys that allow external services and applications to interact with your SaaS Boilerplate account. You can access this page by navigating to `Settings > Organization > API Keys` from the main dashboard sidebar.

![The API Keys page showing API key management tools and existing keys.](/images/help-center/api-keys/01-api-keys-page.png)

This powerful API key management system provides secure access control for integrations, allowing you to connect your SaaS Boilerplate with external tools while maintaining security and monitoring usage.

## API Key Management Features

### **Key Information Display**

Each API key shows essential information for identification and management:

**Displayed Information:**

- **Key Name**: Human-readable identifier for the key
- **Key Preview**: Masked version showing only the last few characters
- **Expiration Status**: Whether the key expires and when
- **Creation Date**: When the key was generated
- **Last Used**: Timestamp of last API call (if available)

### **Key Actions**

Each API key provides several management actions:

**Available Actions:**

1. **Show API Key**: Reveal the full key for copying
2. **Copy API Key**: Copy the key to clipboard
3. **Delete Key**: Remove the key permanently
4. **Edit Key**: Modify key properties (if available)

### **Security Features**

Built-in security measures to protect your API keys:

**Security Measures:**

- **Key Masking**: Keys are displayed partially to prevent shoulder surfing
- **Secure Display**: Full keys are only shown when explicitly requested
- **Access Control**: Only authorized users can manage API keys
- **Audit Trail**: Track key usage and modifications

## Creating New API Keys

### **Key Creation Process**

Follow these steps to create a new API key:

1. **Click Create API Key**: Use the button in the top-right corner
2. **Configure Key Settings**: Set name, permissions, and expiration
3. **Generate Key**: System creates a secure, unique key
4. **Copy and Store**: Securely copy the key for immediate use
5. **Save Configuration**: Store key details for future reference

### **Key Configuration Options**

When creating a new API key, you can configure:

**Basic Settings:**

- **Key Name**: Descriptive name for identification
- **Description**: Optional details about the key's purpose
- **Permissions**: Specific access levels and scopes
- **Expiration**: Optional expiration date for security

**Advanced Settings:**

- **Rate Limiting**: API call frequency restrictions
- **IP Restrictions**: Limit access to specific IP addresses
- **Event Types**: Specific events the key can access
- **Webhook Integration**: Connect to webhook endpoints

## API Key Best Practices

### **Security Guidelines**

Follow these security best practices for API key management:

**Key Security:**

- **Unique Names**: Use descriptive, unique names for each key
- **Limited Permissions**: Grant only necessary access levels
- **Regular Rotation**: Periodically rotate keys for security
- **Secure Storage**: Store keys securely, never in code repositories
- **Monitor Usage**: Regularly check key usage patterns

**Access Control:**

- **Principle of Least Privilege**: Grant minimal required access
- **Key Separation**: Use different keys for different services
- **Environment Isolation**: Separate keys for development/production
- **User Accountability**: Track who created and uses each key

### **Usage Guidelines**

Best practices for using API keys effectively:

**Integration Usage:**

- **Service-Specific Keys**: Create dedicated keys for each integration
- **Documentation**: Document key purpose and usage
- **Testing**: Test integrations with dedicated test keys
- **Monitoring**: Track API usage and performance
- **Error Handling**: Implement proper error handling for API calls
